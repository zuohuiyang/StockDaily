# 数据口径与执行策略

## 1. 市场收盘与数据可用时间

| 市场 | 交易时间 (北京时间) | 收盘时间 (北京时间) | 推荐拉取时间 |
|---|---|---|---|
| **A 股** | 09:30 - 15:00 | 15:00 | 15:30 后 |
| **美股** | 09:30 - 16:00 (美东) | 次日 04:00/05:00 | 次日 06:00 后 |
| **加密货币** | 7x24 小时 | 无 (按日结) | 次日 06:00 后 |

## 2. 日报数据获取策略

### 2.1 智能拉取 (Smart Ingest)
执行 `stock.ingest public-daily` 时，系统会自动检查并补全以下关键日期的数据：
1.  **目标日报日期 (Target Date)**：默认为昨日。如果数据缺失，会自动拉取。如果目标日为非交易日（如周六），系统会自动回退检查并拉取最近交易日（过去 5 天）的数据。
2.  **上一交易日 (Previous Day)**：用于计算日环比。系统会自动检查并补全。
3.  **年初 (Year Start)**：用于计算年累计收益。系统会自动检查并补全（通常为 1月1日 至 1月5日）。

### 2.2 先查库后拉取 (Check-then-Fetch)
为了提高效率并减少网络请求，系统遵循“先查库后拉取”原则：
- 在发起网络请求前，先查询本地数据库是否已有对应日期、对应标的的收盘价。
- 如果数据已存在，跳过拉取（除非使用 `--force` 参数强制刷新）。
- 仅对缺失的数据发起 API 请求。

### 2.3 数据回退 (Fallback)
当生成日报时，如果目标日期的某个标的缺失收盘价（例如休市、停牌）：
- 系统会自动查找该标的在 **最近 5 天内** 的最新收盘价。
- 使用该“回退价格”进行估值计算。
- 在生成的日报中，会显式标记该标的使用了回退数据（价格旁标注 `*`，并在底部列出实际使用的日期）。

## 3. 常见场景示例

**假设当前时间为：2026-03-07 (周六) 10:00**

1.  **生成周五 (03-06) 日报**：
    - 命令：`stock.ingest public-daily --date 2026-03-06`
    - 结果：周五是交易日，数据完整。生成正常日报。

2.  **生成周六 (03-07) 日报**：
    - 命令：`stock.ingest public-daily --date 2026-03-07`
    - 结果：
        - 系统发现周六 A股/美股 休市，自动检查并拉取周五 (03-06) 数据。
        - 生成日报时，A股/美股 使用周五收盘价（标记为回退）。
        - Crypto 使用周六日结价（如果已过 06:00）。
