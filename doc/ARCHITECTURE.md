# 分层架构（草案）

这份文档用来沉淀本项目的“分层与职责边界”，方便后续逐步补充实现细节与改造计划。它可以作为后续 README 的主体结构，未来也可以直接合并/替换到 `README.md`。

## 总览

目标：输入你的**私有账本数据**（持仓、逐笔交易、成本等）与**公网行情数据**（股票/汇率价格），每日生成一份可落地的本地日报（再由你手动或 OpenClaw 读取转发）。

分层（自下而上）：

1. 数据层（事实存储）
2. 数据采集层（公网行情源 + 私有账本源）
3. 业务计算层（估值与盈亏）
4. 交付层（报告落地与触达）

## 第 1 层：数据层（事实存储）

职责：用统一的数据库与表结构保存“可追溯的事实数据”，提供稳定的读写接口（SQL）。  
典型数据：持仓、逐笔交易、每日价格、每日汇率、数据来源标记、必要的时间戳等。

## 第 2 层：数据采集层（Data Ingestion）

职责：把“外部数据/输入数据”变成可入库的记录，并写入数据层；包含必要的数据校验、去重、缺失处理与来源标记。  
该层拆为两部分：

- 公网行情源：来自公开/第三方接口的股票价格、汇率等市场数据（market data）。
- 私有账本源：你维护的持仓、逐笔交易、手续费、分红等账本记录（ledger/book）。

## 第 3 层：业务计算层（估值与盈亏）

职责：基于数据层的事实数据完成组合估值与统计口径的计算，并产出“可展示的结构化结果”。  
典型能力：市值换汇、收益对比（今日/昨日/年初）、逐笔交易成本核算（如 FIFO/加权平均）、已实现/未实现盈亏等。

## 第 4 层：交付层（报告落地与触达）

职责：将业务计算层的结果渲染成日报（Markdown/文本）并写入固定路径，便于你手动查看或让 OpenClaw 读取转发。  
典型产物：`daily_reports/YYYY-MM-DD_report.md`（归档）与 `daily_reports/latest_report.md`（最新）。

## 术语与命名

- 公网行情源：股票/汇率价格等公开数据。
- 私有账本源：持仓、逐笔交易等个人数据。
- 数据采集层：负责把两类数据采集/整理后写入数据库。

## 现有代码如何映射到分层（便于迁移时归类）

- 数据层：`portfolio.db`、`stock_tracker.db` 等 SQLite 文件与表结构约定。
- 数据采集层：抓价/拉汇率/补历史/修复缺失的脚本（把数据写入 DB）。
- 业务计算层：组合估值、对比、成本与盈亏计算相关脚本与函数。
- 交付层：报告生成（stdout/写文件）、调度脚本、本地定时触发与“latest 文件”维护逻辑。
